# LIVE LIFE 需求分析

状态：当前开发基准  
最后更新：2026-06-08

## 1. 产品定位

LIVE LIFE 当前先做成一个东京音乐入口，不是完整电商，也不是单纯活动海报页。

它要同时承担四件事：

- 展示 LIVE LIFE 自己的演出。
- 展示 LIVE LIFE 推荐或整理的演出情报。
- 展示 CD 严选，内部包含 CD 和黑胶。
- 承接票务、外部购买、发货、合作、投稿等联系问题。

一句话版本：

```text
LIVE LIFE 是东京现场、CD/黑胶严选、音乐档案和联系入口。
```

## 2. 固定品牌要求

- 所有前端展示品牌统一写作 `LIVE LIFE`。
- 对外 API 返回的展示字段也统一使用 `LIVE LIFE`。
- 内部代码变量名暂时不强制全部重命名，避免为了命名改动制造额外风险。
- 英文界面里的主要导航和入口文案使用全大写，例如 `SHOWS / CD SELECT / ARCHIVE / CONNECT`。

## 3. 页面结构

当前顶层入口固定为：

```text
Shows / CD 严选 / Archive / Connect
```

三语言对应：

```text
中文：演出情报 / CD 严选 / 档案 / 联系
日本語：ライブ情報 / CDセレクト / アーカイブ / 問い合わせ
English：SHOWS / CD SELECT / ARCHIVE / CONNECT
```

不再设置顶层 `Shop` 页面。

原因：

- 演出票务可能由外部票站代理，流程和站内商品购买不同。
- CD/黑胶未来可能链接 BASE 等外部 Shop，不一定在 LIVE LIFE 站内付款。
- 顶层放 Shop 会让用户误以为 LIVE LIFE 当前已经有完整站内交易系统。

## 4. Shows 需求

Shows 页面包含两类内容：

- LIVE LIFE 自主演出。
- LIVE LIFE 推荐或整理的其他演出、历史活动、视觉档案。

展示规则：

- LIVE LIFE 自己的演出必须固定在最上方。
- 推荐演出和历史档案放在下方。
- 票务链接可以是外部链接，暂时不要求站内结算。
- 如果票务链接未确认，可以先展示“票务待确认”。

当前重点活动：

- 7月10日、7月14日两场 `紅髪少年殺人事件` 演出。
- 7月14日下北泽场共演包含 `ルサンチマン` 和 `おそロシア革命`。

## 5. CD 严选需求

`CD 严选` 是商业入口，但表达上不是普通电商首页，而是 LIVE LIFE 的精选目录。

分类固定为：

```text
CD / 黑胶
```

购买路径：

```text
CD 严选列表 -> 单品详情卡片 -> 点击此处购买 -> 外部 Shop，例如 BASE
```

当前阶段：

- 先不做购物车。
- 先不做站内支付。
- 先不做订单系统。
- 单品可以先用占位数据。
- 购买按钮可以先跳到 BASE 或其他外部 Shop。

## 6. Archive 需求

Archive 用于整理 LIVE LIFE 的历史内容和公开资料。

可包含：

- 过往活动海报。
- 公开信息备注。
- 演出照片。
- 推荐文章。
- 音乐场景资料。

Archive 不应该和即将发生的演出混在一起。

## 7. Connect 需求

原本 `Join us` 的方向改为 `Connect`。

Connect 是统一联系入口，可以承接：

- 票务问题。
- 外部平台付款后没有收到货。
- CD/黑胶购买咨询。
- 发货问题。
- 活动合作。
- 投稿。

当前阶段 Connect 只做本地 API 验证，后续可以接：

- 邮件通知。
- 数据库记录。
- 客服系统。
- 管理后台。

## 8. 三语言需求

默认语言：

```text
中文
```

支持语言：

```text
中文 / 日本語 / English
```

要求：

- 默认显示中文。
- 日本语和英语界面必须翻译，不只替换按钮。
- 英语导航和主要入口使用全大写。
- API 保留三语言字段，前端根据当前语言选择展示。

## 9. 登录注册与支付

当前结论：先暂缓。

原因：

- 如果 CD/黑胶跳 BASE 等外部 Shop，LIVE LIFE 站内不一定需要账户。
- 如果演出票务走外部票站，站内账户也不一定能解决票务问题。
- 如果未来决定站内支付，就必须补订单、支付状态、发货状态、售后入口和用户身份。

审批点：

- 是否做登录注册，等购买路径和票务路径更明确后再决定。

## 10. 设计版本策略

未来会给客户 Review 多版页面设计。

要求：

- V2 先保留。
- V3 已审批通过。
- 未来 V4/V5 可以继续加。
- 右上角增加版本下拉框，例如 `V2 / V3 / V4`。
- 下拉框只用于 Review 或本地预览，不一定在正式上线页面显示。

核心原则：

```text
设计版本可以变化，但需求和后端 API 逻辑先保持稳定。
```

## 11. 当前已审批结论

- 顶部入口：Shows / CD 严选 / Archive / Connect。
- 不设置顶层 Shop。
- CD 严选分为 CD / 黑胶。
- 单品购买按钮跳外部 Shop，例如 BASE。
- Connect 替代 Join us。
- 三语言为中文、日本語、English，默认中文。
- V2 保留。
- V3 通过，方向为经典摇滚乐队名密集纹理、LIVE LIFE 字母轮播、抽象音轨背景。
